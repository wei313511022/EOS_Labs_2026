RTES913 - EOS - Raspberry Pi 4

Lab 5: Kernel Configuration and Performance Tuning

Make one safe, explainable kernel configuration change, then justify and use a public Raspberry Pi benchmark to measure its effect.

Before you begin

Keep a working boot path. This lab builds a new kernel image, not a replacement for the Raspberry Pi OS image. Back up the boot configuration before selecting your Lab 5 image.

Before starting, make sure you have:

  • completed Labs 1 through 4, including a working Lab 4 server that can run on the Raspberry Pi;
  • a dedicated clean rpi-6.1.y Raspberry Pi 4 / 64-bit kernel tree and the arm64 cross-compilation toolchain;
  • a known-good Raspberry Pi OS boot image, SSH access, and an SD-card recovery path; and

By the end of this lab, you will be able to trace a kernel configuration from a default configuration to a booted image, compare two controlled benchmark runs, and explain both the benefit and the cost of your change.

QA

Answer all four questions in your submission. These are open questions; support each answer with the relevant implementation, configuration, benchmark, or demonstration evidence.

  1. Based on Labs 1 to 4, how would you modify .config and make the game performance better? Research and install a public benchmark on your Raspberry Pi. You must decide which benchmark is appropriate, explain why its metrics evaluate your proposed change, and use repeatable baseline and modified-kernel results to prove your modification.
  2. Based on Lab 1 to Lab 4, which part of the project design is weak? Help us to fix it.
  3. Use your creativity to extend a new mechanism or function in this game. The extension must be related to what you learned from the courses.
  4. Do you have any advice to help us make the labs better?
