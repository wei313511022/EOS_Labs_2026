The through-line

One course project.
Every layer connected.

In 2026, RTES913: EOS shifts from isolated implementation drills to AI-assisted system architecture and verification. Students still build real software, but each lab asks them to connect requirements, interfaces, tests, and runtime behavior into one working project.

Overbug is that project: a cooperative classroom game set in the 913 lab. Players help the TAs debug and repair broken Raspberry Pi boards so the lab can keep running and everyone can get back to research on time.

By the end Students can trace a button press from a GPIO pin to a live multiplayer game state.

That is the system they build, test, and explain across the labs.

Round flow

  1. Read the order cards to see which components are needed.
  2. Take a faulty board from the intake area.
  3. Collect resistors, capacitors, and inductors from the bins.
  4. Place the board and components at a soldering workstation.
  5. Hold the solder key near the workstation to attach the component.
  6. Send the soldered board to an inspection station.
  7. Ship a board that passes inspection to score before time runs out.

Basic rules

  • Each player can carry only one item at a time.
  • Each order needs 1 to 4 total components.
  • Each component type can require 0 to 2 pieces.
  • A soldering station holds one board and one pending component.
  • Soldering takes about 2 seconds while the solder key is held.
  • Soldering pauses if the player leaves or releases the key.
  • Inspection is automatic and takes about 4 seconds.
  • Only boards matching an order can be shipped successfully.
  • Wrong or unwanted unlocked boards can be recycled without losing points; a passed board must ship.
